## 3. 系统设计文档（v2.1）

### 3.1 概述

本系统为个人本地化运动数据分析与 AI 教练工具，自动同步佳明中国区账户的运动数据（聚焦跑步，力量训练为可选获取），基于可配置的大语言模型（LLM），通过**多智能体（Multi‑Agent）头脑风暴**生成专业训练报告，并支持基于历史数据的智能问答。所有数据、配置和对话记录完全本地化存储，保障隐私。

### 3.2 架构设计

采用 **B/S 双层本地架构**，通过浏览器访问本地后端服务。
[浏览器：HTML/JS] — HTTP (localhost) → [Python Flask 后端] — HTTPS → [Garmin China API]
│ — HTTPS → [LLM API (DeepSeek v4 等)]
│
[本地 SQLite + 静态文件]


- **前端**：纯 HTML/CSS/JS，使用 Chart.js 可视化，通过 AJAX 与后端 API 通信。
- **后端**：Python 3 + Flask，集成 `garminconnect` 库，调用 LLM 并管理多智能体对话，内置 Web 服务器仅绑定 `127.0.0.1`。
- **数据存储**：SQLite 保存活动摘要、对话历史、配置；原始活动详情 JSON 可按需保存到文件系统。

### 3.3 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Flask | 轻量、易扩展，支持 SSE |
| 佳明数据源 | garminconnect (非官方库) | 中国区域名 `garmin.cn` 登录 |
| LLM 接入 | OpenAI Python SDK (兼容模式) | 支持 DeepSeek v4 (base_url: `https://api.deepseek.com/v1`) |
| 数据库 | SQLite | 单文件，无需额外服务 |
| 前端图表 | Chart.js | CDN 引入，避免本地依赖 |
| 报告 PDF | 浏览器打印 + CSS | 保存为 PDF 由用户操作 |
| 打包 | PyInstaller | 生成单文件夹或单文件执行程序 |

### 3.4 功能模块

#### 3.4.1 佳明账户连接与数据同步
- 用户在前端输入佳明中国区账号、密码（仅本地传输）。
- 后端调用 `garminconnect` 登录，获取 session 并加密保存。
- 支持 **手动同步** 与 **定时自动同步**（可选），增量获取活动列表，仅下载新增活动。
- 活动类型默认获取所有，前端主要展示跑步相关数据；力量训练数据为**可选获取**，若存在则纳入分析，若未同步则不影响核心功能。
- 同步时显示进度条。

#### 3.4.2 运动数据展示
- **仪表盘**：总跑量、月度趋势、配速分布、跑力指数（VDOT）估算等。
- **活动列表**：可按类型（跑步/力量/其他）、日期范围筛选，关键字搜索活动名称。
- **活动详情**：跑步显示配速、心率、步频、海拔等曲线；若有力量数据则显示训练量摘要。
- 图表使用 Chart.js 动态渲染。

#### 3.4.3 AI 训练报告（多智能体头脑风暴）
**目标**  
生成包含四大部分的专业报告：训练数据统计分析、跑步能力趋势、训练建议、成绩预测。

**多智能体流程**  
后端模拟 3 个核心角色（力量专家为附加角色，仅在存在力量数据时激活），通过同一 LLM 接口顺序调用，进行 **3～5 轮对话**：

| 角色 | 系统提示词 |
|------|------------|
| **数据分析师** | 运动科学背景，擅长从数据中提取统计规律、异常检测。 |
| **跑步教练** | 实战派教练，注重周期化训练、配速策略、伤病预防。 |
| **力量训练专家**（可选） | 体能专家，仅在用户有力量数据时参与讨论。 |
| **主教练（总结者）** | 整合多方意见，形成最终报告。 |

**执行步骤**：
1. **数据准备**：提取所选时间区间内的跑步数据摘要（跑量、强度分布、心率区间等），若力量数据存在则一并提取。
2. **首轮发言**：数据分析师给出《训练数据统计分析》初稿。
3. **多轮交叉讨论**：跑步教练基于分析师输出提出能力趋势与训练建议；若力量专家激活，则对力量训练转化效果给予补充或挑战；分析师可再修正。每轮均包含完整上下文。
4. **主教练汇总**：整合所有发言，输出最终报告（含成绩预测），Markdown 格式。
5. **前端渲染**：后端返回报告，前端使用 `marked.js` 渲染，支持打印/保存 PDF。

**可控参数**：
- 轮数可在配置中设置（默认 4 轮）。
- 每轮最大 token 限制。
- 前端显示各 Agent 发言进度。

#### 3.4.4 智能问答
- 用户提问，后端注入近期训练数据摘要（最近 12 周）与最近 5 轮对话历史，调用 LLM 生成回答。
- **流式输出**：通过 Server‑Sent Events (SSE) 逐字返回。
- **对话管理**：
  - 自动保存至 `chat_history` 表。
  - 历史列表支持 **关键词搜索** 和 **单条删除** 或 **清空全部**。
  - 点击历史条目可继续追问。
- 系统提示词固定为“你是私人跑步教练，可根据用户提供的训练数据给出个性化建议”。

#### 3.4.5 配置管理
- 独立设置页，可修改：
  - LLM 服务地址（如 `https://api.deepseek.com/v1`）
  - API Key
  - 模型名称（如 `deepseek-chat`）
  - 多 Agent 对话轮数（3～5）
- 佳明账号不存储密码，仅保存加密 session；可登出清除。
- 所有配置存储于 SQLite `config` 表。

### 3.5 数据模型（SQLite）

```sql
CREATE TABLE activities (
    id INTEGER PRIMARY KEY,              -- Garmin activity ID
    name TEXT,
    type TEXT,                           -- 'running','strength_training', ...
    start_time DATETIME,
    duration REAL,                       -- 秒
    distance REAL,                       -- 米
    avg_heart_rate REAL,
    max_heart_rate REAL,
    avg_pace REAL,                       -- 秒/公里
    elevation_gain REAL,
    raw_json TEXT                        -- 原始活动详情 JSON（力量训练组次数据包含在内）
);

CREATE TABLE chat_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    role TEXT,                           -- 'user' / 'assistant'
    content TEXT,
    context_snapshot TEXT                -- 对话时注入的训练数据摘要
);

CREATE TABLE config (
    key TEXT PRIMARY KEY,
    value TEXT
);

3.6 API 设计
端点	方法	说明
/api/auth/login	POST	佳明登录，返回状态；需验证码时返回提示
/api/sync	POST	触发增量同步，返回进度
/api/activities	GET	查询活动列表，支持 ?type=running&from=2026-01-01&to=2026-05-01&q=关键词
/api/activity/<id>	GET	单个活动详情
/api/stats	GET	聚合统计，支持 ?period=weekly/monthly/yearly
/api/report/generate	POST	启动多 Agent 报告生成，返回 task_id
/api/report/status/<task_id>	GET	查询生成进度（当前轮次、角色）
/api/report/result/<task_id>	GET	获取完成的 Markdown 报告
/api/chat/ask	POST	提问，返回 SSE 流
/api/chat/history	GET	历史对话列表，支持 ?q=搜索词
/api/chat/delete/<id>	DELETE	删除单条对话
/api/chat/clear	DELETE	清空全部对话
/api/config	GET/POST	读取/更新系统配置

#activities：活动列表+详情

#report：报告生成与历史报告查看

#chat：问答界面

UI 风格：简洁卡片式，适配桌面端。

3.8 安全与隐私
后端仅监听 127.0.0.1。

LLM API Key 仅后端可见，前端编辑时通过 POST 传输，不缓存在浏览器。

佳明密码使用后即丢弃，session token 加密存储。

调用 LLM 时仅发送分析必需的摘要数据，不上传完整原始数据。

3.9 打包与分发
使用 PyInstaller 将 Flask 应用打包成单文件或单文件夹可执行程序。

前端静态文件通过 --add-data 参数打入包内。

启动时自动打开默认浏览器至 http://127.0.0.1:5000。

数据库文件放置在可执行文件同级目录（或用户目录），保证持久化。

可提供简单的批处理/shell 脚本作为启动器。

3.10 关键实现细节
佳明中国区适配
garminconnect 库初始化时设置 domain='cn'。如遇验证码，后端返回需要手动输入，前端弹出窗口，用户提交后继续登录流程。

多智能体对话实现
使用 Python 异步生成器，每轮调用 LLM 后将角色发言存入对话列表，前端通过轮询 /api/report/status 或 SSE 获得实时推送。报告生成期间显示各 Agent 名称与当前状态。

力量训练数据为可选
同步时默认下载所有活动类型（包括力量），但用户也可在设置中关闭“获取力量训练”以忽略此类活动。

在报告生成前，检测是否存在力量训练数据。若无，则力量专家角色不激活，报告完全聚焦跑步；若有，则激活并分析其对跑步的辅助效果。

力量数据不完整的（如缺少组重量）系统会进行健壮处理，防止报告生成报错。

成绩预测
LLM 依据跑力（VDOT）、近期间歇配速、训练负荷趋势，参照杰克·丹尼尔斯公式或类似方法估算 5k/10k/半马成绩。提示词中明确要求“基于数据推算，注明假设和不确定性”。

3.12 风险与缓解
风险	缓解措施
garminconnect 库不稳定	锁定版本，备选手动 cookie 导入方案
多 Agent 对话 token 消耗大	压缩训练摘要、允许用户配置廉价模型，轮数可调
力量数据格式差异	精确解析前做格式检测，缺字段时跳过力量相关分析
打包体积过大	使用 --onefile，剔除无用依赖，或改用 Nuitka
