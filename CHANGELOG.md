# Changelog

本文档记录 Running AI Coach 的版本修改历史。

---

## v0.3.0 (2026-06-14)

### 新增功能

#### 训练趋势页面
- 新增 `#trends` 训练趋势页面，查看指定时间段内的训练能力、跑量、配速、心率与恢复趋势
- 顶部 KPI：VDOT、VO2max 估算、周均跑量、平均配速、平均心率、恢复指数
- 四张核心图表：跑力趋势、跑量趋势、配速/心率趋势、恢复趋势
- 周期汇总表格（按时间倒序排列）
- 快捷筛选：近30天、近3个月、今年、全部
- 粒度选择：按天/周/月聚合

#### 跑步热力图
- 新增 `#heatmap` 跑步热力图页面，基于跑步路线坐标显示发光轨迹
- 高德地图集成，支持缩放、拖拽
- 光点大小显示热力密度（蓝色低→绿色中→橙色高→红色极高）
- 统计面板：覆盖城市、总里程、最近跑步
- 热门路线列表（横向卡片布局）
- 城市标签浮层显示当前城市
- 路线数据补采功能（从 Garmin 下载 GPX 数据）

#### 天气数据集成
- 活动同步时自动获取天气数据（温度、湿度、风速、天气状况）
- 天气数据存储为摄氏度，历史数据自动转换
- 活动详情页显示天气信息（单行格式）
- AI 问答和训练报告中引用天气数据
- 设置页新增「天气与位置」配置（城市选择）
- 使用 Open-Meteo 公共 API（免费，无需 API Key）

#### AI 问答增强
- 新增「包含力量训练数据」开关
- 自动带上当前日期和当地近3天天气
- 对话历史持久化（localStorage 保存会话 ID）
- 页面切换后自动恢复上次对话

#### 训练报告增强
- 新增快速选择功能（近一周/当月/近三个月/近半年/当年）
- 默认日期范围为当月第一天到今天
- 分段训练自动识别（同日多次记录视为热身/主训练/冷身）

#### 同步功能优化
- 合并健康同步和活动同步（一次同步完成所有数据）
- 两阶段同步：先全量同步条数，有新增再同步明细
- 适配新用户场景

### 优化

#### VO2max 估算逻辑
- 独立 VO2max 估算算法（ACSM + 心率储备法）
- VDOT 与 VO2max 不再强制相等
- Garmin VO2max 优先显示，估算值作为补充
- 缺少心率数据时显示 `-`

#### 仪表盘视觉统一
- 统一卡片文字层级、字号、字重、颜色
- KPI 卡片：标签 12px/700，数值 28px/800
- 健康恢复卡：数值 22px/800
- VDOT 卡片：数值 40px，列表 15px
- 表格字号统一 15px
- 趋势变化使用 CSS 类（.trend-delta）

#### 配速分布改进
- 分栏表格布局（图表 + 表格）
- 表格显示：配速区间、距离、占比
- 填满卡片底部，无留白

#### 最佳5K时间格式
- 从 `25.3min` 改为 `25:18` 格式（MM:SS）

#### 热力图数据修复
- 修复距离计算错误（不再重复累加活动距离）
- 城市自动识别（支持29个中国主要城市）

### 修复

- **训练报告强度分布错误**：提示词增加硬规则，E区间配速不能错误标注为M/T
- **训练报告同日多次训练误判**：分段训练自动识别，不再视为多次独立训练
- **训练报告配速推断错误**：禁止仅凭配速推断糖原消耗或有氧缺失
- **AI问答6月跑量数据错误**：改为显示当月所有活动（不再限制30条）
- **训练报告未包含当日数据**：默认日期范围改为当月第一天到今天
- **热力图无法加载**：配置高德地图 API Key
- **热力图热门路线不显示**：修复 CSS z-index 和定位
- **AI问答对话丢失**：使用 localStorage 持久化会话 ID
- **仪表盘健康数据不跟随筛选**：健康数据 API 支持日期范围参数

## v0.2.0 (2026-05-08)

### 新增功能

#### 跑力 (VDOT)
- 基于 Jack Daniels 公式的 VDOT 自动计算，以最佳 ≥5km 成绩为基准
- VDOT 校正系数 1.065（经验证与 Daniels 标准表一致）
- 不同距离施加可持续强度系数：5K 95%、10K 90%、半马 86%、全马 82%
- 训练配速区间（E/M/T/I/R）基于 `%vVO2max` 精确映射
- 心率区间基于 Karvonen 储备心率法

#### 跑步动态数据
- 佳明活动详情采集：步频、触地时间、垂直振幅、步幅、训练效果、VO2max、乳酸阈值
- 兼容中国区 API 字段名（`averageRunCadence` 等）
- 活动详情页展示全部 8 项动态指标

#### 健康数据
- 每日健康指标同步：HRV、睡眠、身体电量、压力、静息心率、VO2max
- 仪表盘健康卡片：最新值 + 前日对比趋势
- 双轴趋势图：HRV/睡眠 + 静息心率/身体电量

#### 聊天会话隔离
- 多会话独立上下文，互不干扰
- 会话列表管理，支持创建/切换/删除

#### 跑者基础信息
- 年龄、性别、身高、体重、静息心率、最大心率、跑步目标（5K/10K/半马/全马）
- 信息注入 LLM 上下文用于报告和回答

#### 仪表盘增强
- 快捷时间筛选：近一周/一月/三月/半年/一年/所有
- VDOT 卡片（可折叠展开完整详情）

#### 完整 README
- 功能清单、快速开始、API 端点、数据库表、常见问题

### 优化

#### 数据流
- 训练报告和 AI 问答上下文增加：逐条训练明细、跑步动态、健康数据、心率区间表
- 报告生成时发送全量跑步数据给 LLM（不限 60 条）

#### 性能
- `GET /api/activities`：`SELECT *` → 列白名单，排除 `detail_json`（减少 200MB 传输）
- 数据库索引：`idx_activities_type`、`idx_activities_start`

#### UI/UX
- 全局 CSS 变量主题系统（颜色/阴影/圆角/过渡）
- 导航栏品牌标题 "🏃 Running AI Coach"
- 仪表盘紧凑布局：卡片内边距 24→20px，统计卡片字号 28→24px
- 活动详情双列布局（基础指标左 / 跑步动态右）
- Toast 通知 slide-in/slide-out 动画
- 响应式断点（768px / 480px）

#### 同步
- Token 持久化免重复登录（`garmin_tokens`）
- 新增活动自动获取详情数据
- 批量补采接口 `POST /api/sync/backfill`

### 修复

- **API Key 掩码覆盖**：`GET /api/config` 返回掩码 key，保存时跳过掩码值
- **数据库路径漂移**：默认路径改为以 `backend/database.py` 为基准的绝对路径
- **CN patch 恢复**：`_restore_patches()` 正确恢复 mobile login 方法和 `_establish_session`
- **sqlite3.Row 无 `.get()`**：`_get_training_summary` 和 `_get_health_summary` 改用索引访问
- **VDOT 虚高**：最低计算距离从 ≥3km 改为 ≥5km，排除短距离冲刺
- **成绩预测格式**：`predict_time_str` 支持 `h:mm:ss`
- **同 VDOT 配速不一致**：不同距离施加不同强度系数
- **配速区间错误**：`%VO2max` → 正确映射为 `%vVO2max` 分数
- **健康数据路由冲突**：`/api/health` 改为 `/api/ping`（健康检查）
- **静息心率未取到**：中国区 API 返回路径 `allMetrics.metricsMap.WELLNESS_RESTING_HEART_RATE[0].value`
- **LLM 重试**：`llm_client` 增加指数退避重试（429/5xx/连接错误）
- **settings.js 语法错误**：补采按钮 `finally` 块漏 `}`
